# Contributing to Stress Simulator

First off, thank you for considering contributing to Stress Simulator! It's people like you that make this tool better for everyone.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please open an issue on GitHub. Be as descriptive as possible and include:
- A clear and concise description of the bug.
- Steps to reproduce the bug.
- Expected behavior vs. actual behavior.
- Your system environment (OS, Go version).

### Suggesting Enhancements
Have an idea for a new feature? Open an issue and label it as `enhancement`. We'll discuss the feasibility and implementation details there.

### Pull Requests
1. Fork the repository.
2. Create a new branch for your feature or bug fix.
3. Make your changes and ensure the code follows the project's style.
4. Update the README if your changes affect the usage or features.
5. Submit a Pull Request with a clear description of the changes.

## Coding Guidelines

- **Language**: Go 1.26+
- **Style**: Follow standard Go formatting (`gofmt`).
- **Testing**: Please include tests for any new logic or critical bug fixes.
- **Commit Messages**: Use clear, imperative commit messages (e.g., "Fix CPU spike calculation bug").
